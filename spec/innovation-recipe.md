# Reaching non-trivial, innovative output — the recipe (2026-06-05)

> Goal (user): hit the level of innovation seen in real design-thinking project networks (e.g.
> sugar-network.org) — genuinely novel, non-obvious solution concepts, not incremental UX tweaks —
> **via the harness**, reliably, for any methodology. All levers below are DATA/capability, never
> hardcoded design-thinking vocabulary.

## What makes the difference (demonstrated end-to-end on the KFZ→LV showcase)

1. **Honest, deep exploration first.** A few rich MULTI-persona councils across distinct angles +
   a deliberate **disconfirmation council** (load the likely refuters, test a Non-Fit motion). This
   produced the premise-refuting, mechanism-level POV ("it's an EXPERIENCE gap, not an explain gap";
   the literate reject LV *because* they grasp compounding). Innovation needs a sharp, surprising POV
   to aim at — not a shallow brief. (Levers: deep-seed discover wording; `disconfirmation` angle in
   suggestion data; `record_frame` grounding gate.)

2. **Ideation lenses force lateral thinking.** `next_action.act.ideation_lenses`
   (`suggestions/ideation_lenses.json`): cross-domain analogy, **make-the-invisible-EXPERIENCEABLE →
   an interactive model**, provocation/reversal, extreme-user, mechanism-transfer, hybrid,
   subtract-the-obvious, honest anti-goal. The seed intent demands ≥1 *unexpected mechanism*. These
   turned conventional concepts into: a tool that **weaponises the user's own intuition against the
   product** (trust-inversion), an **experienceable relational model**, an **anti-product that says NO
   to 80%**.

3. **An interactive `model` makes the invisible felt.** The new `model` artifact type + `spa-model`
   template (`range`/`number` inputs → `computed`/`bar` with a live `formula`) lets a prototype be a
   STEERABLE simulation, not a static screen. The showcase's winning concept — "Der volle Stuhl",
   where you steer your income/shock and watch your livelihood hold or tear — is exactly the class of
   experiential prototype that reads as innovative. This is usually the single biggest move.

4. **Diverse KINDS, not N forms.** `next_action.act.artifact_palette` surfaces the archetypes
   (flow/comparison/dashboard/cards/**model**) from data; pick across them + ≥1 dark-horse.

5. **Make innovation reliable, not luck.** `assess_project.novelty` reports the distinct artifact
   kinds + whether an interactive model exists, and flags a "narrow" (forms-only) space as a gap —
   so a run pushes for a bolder, more experienceable concept instead of settling.

6. **Validate the bold concept for real.** Build it as a real `model`, drive it via Playwright, and
   record **grounded** proband sessions across segments (incl. the skeptic) — the trust-inversion was
   confirmed grounded (Niklas trusts it *because it gives him right and doesn't sell*). Groundedness
   is now enforced (the log survives `proto_close`; an unverified session won't satisfy a gate).

7. **The answer renders rich + connected.** Structured convergence blocks (clusters/key_problems/
   ranking/shortlist) persist + render (GAP-3); the diamonds connect via the synthesis spine (GAP-6);
   section overlays organize it. The result is explorable and reads as a real project.

## Recipe in one line
Deep + disconfirming exploration → a sharp surprising POV → **lenses → ≥1 unexpected mechanism, ≥1
experienceable interactive model, ≥1 dark-horse** → varied KINDS (novelty signal guards against
forms-only) → build the bold model for real and **ground**-test it across segments → a rich,
connected, honestly-non-targeted deliverable.

All of this is methodology-agnostic data/capability; an invented methodology gets the same levers.
